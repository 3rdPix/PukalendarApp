Módulo de calificaciones

La nota mínima en la universidad es $1.0$, la nota de aprobación es $4.0$, y la nota máxima es $7.0$. La exigencia refiere al porcentaje de correctitud que debe tener una evaluación para alcanzar la nota de aprobación, de este modo, en una prueba con $100$ preguntas al $60\%$ de exigencia, se deben tener justamente $60$ respuestas correctas para alcanzar la nota $4.0$.

El formato de cálculo más simple implica crear dos rectas; una para el segmento llamado "rojo" (bajo la nota de aprobación) entre $[1.0, 4.0)$, y otra para el segmento llamado "azul" (sobre la nota de aprobación) entre $[4.0, 7.0]$.

Luego, si se tiene $x$ porcentaje de respuestas correctas para una evaluación, la nota será determinada según:

  • Para rojos: ($x < \text{exigencia}$) $$ y = \left( \frac{7.0 - 4.0}{\text{exigencia}} \right) x + 4.0 $$

  • Para azules: ($x \geq \text{exigencia}$)

$$ y = \left( \frac{7.0 -4.0}{100.0 - \text{exigencia}} \right) x + 4.0 - \left( \frac{7.0 -4.0}{100.0 - \text{exigencia}} \right) \text{exigencia} $$

 71def calculate_blue(score: float,
 72                   threshold: int, add_base: bool) -> float:
 73    slope = (GRADE_MAXIMUM - GRADE_PASSING) / (100 - threshold)
 74    slope_based = ((GRADE_MAXIMUM_FOR_BASED - GRADE_PASSING_FOR_BASED) 
 75                   / (100 - threshold))
 76    if not add_base:
 77        grade = slope * score - slope * threshold + GRADE_PASSING
 78        return grade
 79    grade = (slope_based * score - slope_based * threshold
 80             + GRADE_PASSING_FOR_BASED + 1)
 81    return grade    
 82
 83
 84def calculate_red(score: float, threshold: int, add_base: bool) -> float:
 85    slope = (GRADE_PASSING - GRADE_MINIMUM) / threshold
 86    slope_based = (GRADE_PASSING_FOR_BASED - GRADE_MINIMUM_FOR_BASED) / threshold
 87    if not add_base:
 88        grade = slope * score + GRADE_MINIMUM
 89        return grade
 90    grade = slope_based * score + GRADE_MINIMUM_FOR_BASED
 91    return grade
 92
 93
 94def calculate_grade(obtained_score: float, total_score: float,
 95                    threshold: int, add_base: bool) -> float:
 96    percentage = 100 * (obtained_score / total_score)
 97    if percentage >= 60:
 98        return calculate_blue(percentage, threshold, add_base)
 99    else:
100        return calculate_red(percentage, threshold, add_base)

103class Grade(ABC):
104    """
105    Nota
106    ----
107    La unidad atómica de calificación. Por si sola no tiene sentido, es
108    necesario unirla por medio de ponderaciones u ecuaciones a otras para
109    formar algún promedio.
110
111    + `obtained_score`: el puntaje obtenido de un total determinado, si no
112    se especifica `total_score` se asume que se trata de un porcentaje de
113    respuestas correctas.
114
115    + `total_score`: el puntaje máximo obtenible. Por ejemplo, en una prueba
116    de 80 preguntas de alternativa, el puntaje máximo obtenible es 80.
117
118    + `threshold`: porcentaje de respuestas correctas necesarias para obtener
119    el mínimo grado de aprobación. Por defecto es 60.
120
121    + `obtained_grade`: sobrescribe la nota calculada según `obtained_score`.
122    Debe estar entre 1.0 y 7.0.
123
124    + `add_base`: si se habilita, se le agrega un punto base (1.0) a la nota
125    calculada según el puntaje obtenido.
126
127    Esta estructura lleva nombre (puede ser definido por el usuario).
128    """
129    def _get_grade(self) -> float:
130        if hasattr(self, "_obtained_grade"): return self._obtained_grade
131        if hasattr(self, "_mathed_grade"): return self._mathed_grade
132        return -1
133    def _set_grade(self, forced_value: float) -> None:
134        if not GRADE_MINIMUM <= forced_value <= GRADE_MAXIMUM:
135            raise ValueError("Obtained grade not within valid range")
136        self._obtained_grade = forced_value
137    value = property(_get_grade, _set_grade)
138
139    def assign_numbers(self, obtained_score: Optional[float]=None,
140                       total_score: float=100.0,
141                       threshold: int=DEFAULT_THRESHOLD,
142                       obtained_grade: Optional[float]=None,
143                       add_base: bool=False) -> None:
144        if obtained_score is not None:
145            condicion_1 = obtained_score >= 0
146            condicion_2 = obtained_score <= total_score
147            condicion_3 = total_score > 0
148            condicion_4 = 0 < threshold < 100
149            if not all([condicion_1, condicion_2, condicion_3, condicion_4]):
150                raise ValueError("Conditions not met to calculate grade")
151            self._mathed_grade = calculate_grade(obtained_score, total_score,
152                                                 threshold, add_base)
153        if obtained_grade:
154            if not GRADE_MINIMUM <= obtained_grade <= GRADE_MAXIMUM:
155                raise ValueError("Obtained grade not within valid range")
156            self._obtained_grade = obtained_grade
157

164    @staticmethod
165    def arithmetic_mean(grades: list[Grade]) -> float:
166        return sum(grade.value for grade in grades) / len(grades)
167
168    @staticmethod
169    def geometric_mean(grades: list[Grade]) -> float:
170        return prod(grade.value for grade in grades) ** (1 / len(grades))
171
172    @staticmethod
173    def harmonic_mean(grades: list[Grade]) -> float:
174        return len(grades) / sum(1 / grade.value for grade in grades)

177class GradeSimple(Grade):
178    """
179    Calificación simple
180    -------------------
181    La estructura de notas más sencilla para un ramo está conformada por
182    ponderaciones de caalificaciones directas que pertenecen a esta clase.
183    Esta estructura además acepta la posibilidad de integrar puntos en
184    forma de \"décimas\" que son agregadas a la nota.
185
186    Un ejemplo de un ramo que utilice solo este tipo de calificación puede
187    ser como el siguiente:
188
189    **Lenguaje Rúnico II**
190      - Interrogación 1 (40%): 7.0 * (0.4) = 2.8
191      - Interrogación 2 (40%) (+2 dećimas): 6.0 * (0.4) + 0.2 = 2.6
192      - Examen (20%): 5.0 * (0.2) = 1
193      Nota final: 6.4
194    
195    En este escenario, cada evaluación (Interrogación 1, Interrogación 2, y
196    Examen) es una estructura de tipo `GradeSimple` con su propia ponderación.
197
198    + `name`: el nombre asociado a la evaluación
199
200    + `ponderation`: el porcentaje de importancia de la evaluación en el
201    conjunto al que pertenece. Es nulo por defecto, de modo que pasa como
202    argumento en el cálculo de media aritmética.
203
204    + `extra_points`: décimas extra que se aplican a la calificación, en caso
205    de que existan.
206
207    + `extremist`: si la evaluación tiene requisitos de rendimiento propias,
208    es decir, requiere de una nota mínima para aprobar el curso, se debe
209    indicar a través de este atributo. Esto ocurre en ocasiones con algunos
210    examenes llamados "reprobatorios" que exigen una nota mínima de 3.95
211    por si propia cuenta para aprobar el ramo.
212    """
213    def __init__(self, id: int, name: str) -> None:
214        super().__init__()
215        self.id: int = id
216        self.name: str = name
217
218    def define_relation(self, ponderator: Optional[int]=None,
219                        extra_points: int=0,
220                        extremist: Optional[float]=None) -> None:
221        if ponderator is not None: self.ponderator = ponderator
222        self.extra_points = extra_points
223        if extremist is not None: self.extremist = extremist
224    
225    def _get_relative_value(self) -> float:
226        concurrent: float = self.value
227        if hasattr(self, "extra_points") \
228           and not hasattr(self, "_obtained_grade"):
229            concurrent += self.extra_points / 10
230        if hasattr(self, "ponderator"):
231            concurrent *= (self.ponderator / 100)
232        return concurrent
233    value_relative = property(_get_relative_value)
234    """
235    Es el valor que toma realmente la evaluación para el aporte final de
236    la nota dentro del grupo al que pertenece. Si no tiene ponderador,
237    retornará su valor natural y el contenedor es el encargado de integrar
238    ese valor a la ecuación adecuada.
239    """

259class GradeGroup(GradeSimple):
260    """
261    Calificación de grupo
262    ---------------------
263    Esta estructura busca agrupar evaluaciones que están asociadas a un mismo
264    ponderador. Por lo general esta toma forma de controles que se realizan
265    a lo largo del semestre. Pero es posible agrupar `GradeSimple` de modo que
266    la nota final de este grupo se calcule como una ponderación de sus
267    elementos. Esto aplica únicamente a aquellos elementos que tengan
268    asignado un porcentaje.  
269
270    Esta estructura, en conjunto con `GradeSimple` debería ser suficiente
271    para modelar la gran mayoría de los cursos. Un curso en sí, es una
272    estructura `GradeGroup` que contiene otros `GradeGroup` dentro de sí, los
273    cuales interactúan con los `GradeSimple` definidos para el curso.
274    """
275    def __init__(self, id: int, name: str) -> None:
276        self.id = id
277        self.name = name
278        self.id_handler: Iterator = count()
279        self._total_ponderation: int = 100
280        self.group_ponderated = dict[int, GradeSimple] = defaultdict()
281        self.group_unponderated = dict[int, GradeSimple] = defaultdict()
282        # Por defecto dejamos la media aritmética
283        self.mathing_formula: GradingFormula = MathingGrades.arithmetic_mean
284    
285    
286    def try_ponderation(self, request: int) -> int:
287        """
288        Asegura que la ponderación total de los elementos no supere el valor
289        máximo del ramo (100%).
290        """
291        if request is None or request < 1: return None
292        if self._total_ponderation < request:
293            return self.try_ponderation(self._total_ponderation)
294        approved = int(request)
295        self._total_ponderation -= request
296        return approved
297
298    # Redefinimos el valor
299    def _get_grade(self) -> float:
300        if hasattr(self, "_obtained_grade"): return self._obtained_grade
301        total_grade = 0
302        for grade in self.group_ponderated.values():
303            total_grade += grade.value_relative
304        lonely_grades = list(self.group_unponderated.values())
305        total_grade += self.mathing_formula.__call__(lonely_grades)
306        # Agregamos un checkeo solo por si acaso
307        if not GRADE_MINIMUM <= total_grade <= GRADE_MAXIMUM:
308            log.error(f"Couldn't grade {self}:{total_grade}")
309            raise ValueError
310        self._mathed_grade = total_grade
311        return total_grade
312    def _set_grade(self, forced_value: float) -> None:
313        if not GRADE_MINIMUM <= forced_value <= GRADE_MAXIMUM:
314            raise ValueError("Obtained grade not within valid range")
315        self._obtained_grade = forced_value
316    value = property(_get_grade, _set_grade)
317
318    def get_eval_from_id(self, id: int) -> object:
319        # Si o si es de este nivel
320        if id in self.group_ponderated.keys():
321            return self.group_ponderated.get(id)
322        elif id in self.group_unponderated.keys():
323            return self.group_unponderated.get(id)
324        else:
325            log.error(f"Tried to find id:{id} at {self} but"
326                      f" wasn't found on: {self.group_ponderated}"
327                      f" or {self.group_unponderated}")
328            raise KeyError("Id not here")

330    def create_grade(self, locate: list[int], requested_type: object,
331                     name: str) -> int|None:
332        """
333        Instancia una evaluación del tipo solicitado y retorna el `id` para
334        referenciarle posteriormente.
335        """
336        # Si soy un group y me llega len 0 es porque yo tengo
337        # que crearlo a mi nivel
338        if len(locate) > 0:
339            next_step: int = locate.pop(0)
340            next_who: GradeGroup = self.get_eval_from_id(next_step)
341            return next_who.create_grade(locate, requested_type, name)
342        # si llego aquí significa que yo tengo que crearlo
343        new: GradeSimple|GradeGroup = requested_type(
344            next(self.id_handler), name)
345        self.group_unponderated.__setitem__(new.id, new)
346        return new.id

349        """
350        Elimina (por completo de la faz) una evaluación
351        """
352        if len(locate) > 1:
353            next_step: int = locate.pop(0)
354            next_who: GradeGroup = self.get_eval_from_id(next_step)

358            self.group_ponderated.__delitem__(who_to_kill)
359        elif who_to_kill in self.group_unponderated.keys():
360            self.group_unponderated.__delitem__(who_to_kill)
361        else:
362            log.error(f"Tried to delete id:{who_to_kill} but wasn't found on"
363                      f" {self.group_ponderated} or {self.group_unponderated}")
364            raise KeyError("Id not here")
365        return True

402class GradeFormatted:
403    """
404    Calificación de formato complejo
405    --------------------------------
406    Estructura grupal que relaciona varias instancias de `Grade` a través de
407    una conexión compleja que no puede ser definida por ponderadores ni
408    combinaciones lineales. Esto incluye grupos dinámicos en que la nota y
409    las evaluaciones que afectan a la calificación están condicionadas por
410    expresiones no simples.
411
412    Por ejemplo: supóngase el ramo **Magia Elemental I** que tiene dos
413    interrogaciones y un examen. Estas evaluaciones forman el grupo "notas
414    teóricas" cuya ponderación en la nota final es del 70%. Sin embargo,
415    dentro de este grupo de notas teóricas la importancia está definida de
416    la siguiente manera:
417
418    >>> i1, i2, Ex = Interrogacion1, Interrogacion2, Examen
419    >>> nota_teorica = (i1 + i2 + 2 * Ex - min(i1, i2, Ex)) / 3
420
421    Es decir, `nota_teorica` es la media aritmética del examen, en conjunto
422    con las siguiente dos notas de mayor valor, pudiendo incluir al mismo
423    examen nuevamente. Luego de este cálculo, el valor de `nota_teorica`
424    es ponderado para considerar su aporte a la nota final del curso.
425    """

427                 *args, **kwargs) -> None: ...

430    def conditional_exprs(self, *args, **kwargs) -> None: ...
431    def explicit_input(self, expr: str, *args, **kwargs): ...
432    def _check_safe(self) -> None: ...
433    def nosequehacer(self) -> True: ...

446    ---------------
447    Cada curso tiene asociada una tabla de grados que contiene todas las
448    sub-estructuras de calificaciones que el usuario defina. Esta clase
449    se encarga de verificar una repartición ponderada coherente, así como

458        self.unassigned: dict[int, Evals] = defaultdict()

462        if request is None or request < 1: return None
463        if self._table_level < request:
464            return self.try_ponderation(self._table_level)
465        self._table_level -= request
466        return request

473                                   ponderation=ponderation,
474                                   **{key: data.get(key) for key in \
475                                      data.keys() if \
476                                      key not in ("name", "ponderation")})
477        elif requested_type == GradeGroup:
478            creation = GradeGroup(next(self._ids), data.get("name"),
479                                  ponderation=ponderation,
480                                  **{key: data.get(key) for key in data.keys()\
481                                     if key not in ("name", "ponderation")})
482        else:
483            raise WtfError("GradeTable>create_grade on grades.py")
484        if creation.ponderation is None:
485            self.unponderated_groups.__setitem__(creation.id, creation)
486        else:
487            self.ponderated_groups.__setitem__(creation.id, creation)
488        return True

500                          f"at level {this_level}->{locate} which is"

La estructura de notas más sencilla para un ramo está conformada por ponderaciones de caalificaciones directas que pertenecen a esta clase. Esta estructura además acepta la posibilidad de integrar puntos en forma de "décimas" que son agregadas a la nota.

Lenguaje Rúnico II

  • Examen (20%): 5.0 * (0.2) = 1 Nota final: 6.4

En este escenario, cada evaluación (Interrogación 1, Interrogación 2, y Examen) es una estructura de tipo GradeSimple con su propia ponderación.

  • extremist: si la evaluación tiene requisitos de rendimiento propias, es decir, requiere de una nota mínima para aprobar el curso, se debe indicar a través de este atributo.
